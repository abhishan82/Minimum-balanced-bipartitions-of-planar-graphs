# Prover Log

One line per proof attempt that gets stuck or fails on a lemma, appended at
the time it happens (see CLAUDE.md). This feeds a planned benchmark paper on
LLM-assisted formalization — failures are data worth recording, not just
successes. An empty log below a given date range is itself meaningful data:
it means no proof attempt was made in that period, not that everything
succeeded on the first try.

Format:

```
date | lemma | model/tool | outcome | notes
```
